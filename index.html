Overview

Thrust vector control is a pure GNC problem: an inherently unstable vehicle must know its own attitude and motion from noisy sensors, and continuously steer its thrust vector to stay stable and on the commanded attitude. This study builds that capability in the order a real program would — model, simulate, then fly the software on hardware.

Phase 1 · Days 01–05

Dynamics & the Plant Model

Coordinate frames, Euler angles, DCMs, quaternions, moments of inertia, and Euler's rotational equations. Ends with an open-loop 3DOF Simulink model of the vehicle.

Phase 2 · Days 06–10

Closing the Loop

PID attitude control, servo/actuator limits, sensor noise models, complementary and Kalman filtering — all running as discrete-time flight software inside the sim.

Flight Software & Hardware

Milestone Reviews

Progress is gated by three reviews modeled on flight-test practice. Each is a short briefing to your instructor: show the artifact, defend the engineering, take questions.

Getting Started

Before Day 1

  • Read the syllabus — outcomes, deliverables, and how the three weeks fit together.
  • Install MATLAB + Simulink with the Aerospace Blockset and Control System Toolbox, and the Arduino IDE.
  • Create a GitHub repository for the study. All code, models, and your engineering notebook live there.
  • Order or draw the hardware — parts should be in hand by the start of Phase 3.
  • Skim the Start here items on the resources page for motivation — especially the BPS.space Scout engineering video.
Ground rules This study is benchtop-only: no live rocket motors are fired. Any future motor testing or flight happens under instructor supervision and the NAR safety code, outside the scope of these three weeks.
