DAY05

Objectives

  • Implement the 3DOF EOM in Simulink — either from integrator blocks/MATLAB Function blocks, or using the Aerospace Blockset 3DOF rigid-body block (both are acceptable; know what the block does internally).
  • Structure the top level cleanly now: Plant (continuous), Actuators, Sensors, Flight Software (empty for now) — this is the architecture the whole course builds on.
  • Verify the model with sanity checks rather than vibes.
Primer This is the day Phase 1's math becomes a running model — and the day you set the architecture the rest of the course inherits. Structure the top level into four subsystems now, even though three of them are nearly empty: Plant (the continuous rigid-body dynamics you derived Day 4), Actuators, Sensors, and Flight Software. Building this separation on day one — instead of one tangled diagram — is what lets Phase 2 insert a controller, sensor noise, and an estimator without a rewrite, and what makes the Phase 3 hardware port an honest like-for-like comparison instead of a fresh reimplementation. Equally important today is the discipline of verification: a model that "looks right" is not the same as a model you've proven right. You'll run three specific checks — vertical ascent with zero gimbal deflection should match a hand-propagated 1D trajectory; a fixed small gimbal angle should produce a constant, calculable angular acceleration while thrust is on; zero thrust should produce a clean ballistic arc. Only after those pass do you run the instability demonstration — a 2° initial offset with no controller, tumbling — which becomes Exhibit A at MR-1 for why the rest of this course exists.

Do

  • Run verification cases: (a) δ = 0, vertical start → altitude profile matches a hand-propagated 1D ascent; (b) fixed δ = 2° → constant angular acceleration ≈ T·δ·L/J while thrusting; (c) no thrust → ballistic arc.
  • Demonstrate the instability: start 2° off vertical with δ = 0 and show the divergence. This plot is Exhibit A at MR-1 for why the vehicle needs a controller.
  • Parameter sweep: how does time-to-tumble change with CG location and moment of inertia?

MR-1 — Plant Model Review

Ready when you can check every box:

  • Block diagram of the GNC loop, refined from Day 1, with frames and units labeled.
  • Hand derivation of the 3DOF EOM, including the TVC torque term and its small-angle linearization.
  • Open-loop Simulink model organized as Plant / Actuators / Sensors / Flight Software.
  • At least three verification cases with plots, including the instability demonstration.
  • Can answer: Why can't Euler angles alone carry us through a 90° pitch-over? What physically sets how fast the vehicle falls over? What happens to controllability as the motor burns out?
