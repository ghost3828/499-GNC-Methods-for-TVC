Phase 01 · Flight days 01–05 · Milestone MR-1

Dynamics & the Plant Model

Before you can control the rocket, you need a mathematical model of it — the plant. This week builds that model from first principles: how attitude is represented in 3D, how torques change it, and how a gimbaled motor produces those torques. It ends with an open-loop Simulink simulation you trust because you derived it.

Phase goal Understand 3D attitude parametrizations and rigid-body equations of motion, and implement them as an open-loop (no controller yet) 3DOF simulation of the vehicle in Simulink.
DAY01

Mission Framing & GNC Architecture

What problem are we solving, and what does a GNC system look like?

Objectives

  • Describe the roles of Guidance (where should I go?), Navigation (where am I and which way am I pointing?), and Control (how do I actuate to follow the plan?).
  • Trace the closed loop: sensors → state estimator → guidance/control laws → actuators → vehicle dynamics → sensors.
  • Explain why a TVC rocket is unstable without active control (thrust behind the CG, no fins doing the work).

Do

  • Watch the BPS.space Scout F engineering video end-to-end; list every subsystem you see and which GNC function it serves.
  • Draw a block diagram of the full GNC loop for a TVC vehicle. Label signals with units (rad, rad/s, m/s², PWM µs). You will refine this diagram all course.
  • Set up tools: MATLAB/Simulink + Aerospace Blockset, Arduino IDE, Git repo with folders /notebook, /sim, /flight-software, /report.

Read / Watch

  • Start here BPS.space — Scout F, Engineering Cut
  • Start here BPS.space — Modeling a Thrust Vectored Rocket in Simulink (skim for where we're headed)

DAY02

Coordinate Frames, Euler Angles & DCMs

Attitude is the relationship between two frames. Get rigorous about which frames, and how to write that relationship down.

Objectives

  • Define the body frame (fixed to the rocket) and the navigation/world/inertial frame (fixed to the launch site), and state which quantities live in which frame (gyro rates: body; gravity: world; thrust: body).
  • Build a rotation matrix (direction cosine matrix, DCM) from a sequence of Euler rotations; verify it is orthonormal (R·Rᵀ = I, det = 1).
  • Demonstrate gimbal lock: find the Euler-angle configuration where a degree of freedom disappears, and explain why that matters for a rocket that may pitch through 90°.

Do

  • Worksheet: given a rocket pitched 20° and yawed 10°, express the gravity vector in the body frame by hand, then confirm in MATLAB.
  • Write MATLAB functions euler2dcm and dcm2euler; round-trip random angles and check the error.

Read / Watch

  • Start here DCM Tutorial — An Introduction to Orientation Kinematics (starlino)
  • Shared folder AFIT MECH-632 attitude slides
  • Wie, Space Vehicle Dynamics and Control — attitude kinematics chapter (course reserve)
DeliverableCompleted frame-rotation worksheet + verified euler2dcm/dcm2euler functions with test script.
DAY03

Quaternions

The attitude representation your flight computer will actually use: no gimbal lock, four numbers, one constraint.

Objectives

  • Interpret a unit quaternion as an axis-angle rotation, and convert quaternion ↔ DCM ↔ Euler angles.
  • State and implement the quaternion kinematic differential equation — how body angular rate ω drives the quaternion, q̇ = ½ q ⊗ [0; ω].
  • Explain why the quaternion must be renormalized during numerical integration.

Do

  • MATLAB script: integrate the quaternion kinematics for a synthetic gyro-rate history (e.g., a 90° pitch-over maneuver); plot the recovered Euler angles and the quaternion norm drift with and without renormalization.
  • Repeat the same maneuver with Euler-angle kinematics and watch what happens near 90° pitch. Write two paragraphs in your notebook on why the quaternion wins.

Read / Watch

  • Wie, Space Vehicle Dynamics and Control — quaternion sections
  • Shared folder AFIT MECH-632 slides — quaternion kinematics
  • Arduino quaternion/orientation library (untested by instructor — evaluate before trusting; you'll revisit it Day 13)
DeliverableQuaternion propagation script with plots; notebook entry comparing Euler vs. quaternion propagation.
DAY04

Rigid-Body Dynamics & the TVC Torque Model

Kinematics said how attitude changes given rates; dynamics says how rates change given torques — and the gimbal is where torque comes from.

Objectives

  • Compute/estimate the rocket's moments of inertia (component buildup in CAD or hand calc; note the bifilar/pendulum swing test as the experimental method for later).
  • Write Euler's rotational equations of motion and explain each term, including why gyroscopic coupling nearly vanishes for a slender, slowly rolling rocket.
  • Derive the TVC moment: gimbal deflection δ tilts the thrust vector T, producing torque ≈ T·sin(δ)·L about the CG, where L is the gimbal-to-CG distance. Identify the small-angle linearization and its limits (typical TVC mounts: ±5–10°).
  • Assemble the planar (pitch-plane) 3DOF equations of motion: states [x, z, θ, ẋ, ż, θ̇], inputs thrust magnitude T(t) and gimbal angle δ(t), plus gravity.

Do

  • Hand-derive the 3DOF EOM start to finish. Check limiting cases: δ = 0 and perfectly vertical → pure 1D ascent; θ ≠ 0 with δ = 0 → gravity turn + tumble tendency.
  • Estimate mass properties for a plausible vehicle (≈600–900 g, ≈65 cm long, F-class motor) and record them as the parameter file vehicle_params.m your sim will load.
  • Pull a real thrust curve (e.g., Estes F15) from thrustcurve.org and save it as a lookup table.

Read / Watch

  • Wie, Space Vehicle Dynamics and Control — rigid body dynamics
  • Shared folder AFIT MECH-632 slides — Euler's equations
DeliverableScanned hand derivation of the 3DOF EOM with limiting-case checks; vehicle_params.m; thrust-curve lookup table.
DAY05

Simulink Bootcamp → Open-Loop 3DOF Plant

Turn yesterday's equations into a running simulation, and prove to yourself it's right before anyone asks you to control it.

Objectives

  • Implement the 3DOF EOM in Simulink — either from integrator blocks/MATLAB Function blocks, or using the Aerospace Blockset 3DOF rigid-body block (both are acceptable; know what the block does internally).
  • Structure the top level cleanly now: Plant (continuous), Actuators, Sensors, Flight Software (empty for now) — this is the architecture the whole course builds on.
  • Verify the model with sanity checks rather than vibes.

Do

  • Run verification cases: (a) δ = 0, vertical start → altitude profile matches a hand-propagated 1D ascent; (b) fixed δ = 2° → constant angular acceleration ≈ T·δ·L/J while thrusting; (c) no thrust → ballistic arc.
  • Demonstrate the instability: start 2° off vertical with δ = 0 and show the divergence. This plot is Exhibit A at MR-1 for why the vehicle needs a controller.
  • Parameter sweep: how does time-to-tumble change with CG location and moment of inertia?

Read / Watch

  • Shared folder MATLAB Review slides
  • MathWorks — Modeling a TVC'd Rocket in Simulink (reference architecture; don't copy, compare)
  • BPS.space Scout F — simulation section

Milestone Review · Day 05

MR-1 — Plant Model Review

A 20–30 minute briefing to the instructor. Ready when you can check every box:

  • Block diagram of the GNC loop, refined from Day 1, with frames and units labeled.
  • Hand derivation of the 3DOF EOM, including the TVC torque term and its small-angle linearization.
  • Open-loop Simulink model organized as Plant / Actuators / Sensors / Flight Software.
  • At least three verification cases with plots, including the instability demonstration.
  • Can answer: Why can't Euler angles alone carry us through a 90° pitch-over? What physically sets how fast the vehicle falls over? What happens to controllability as the motor burns out?
