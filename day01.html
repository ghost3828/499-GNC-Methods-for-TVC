Mission Framing & GNC Architecture

What problem are we solving, and what does a GNC system look like?

DAY01

Objectives

  • Describe the roles of Guidance (where should I go?), Navigation (where am I and which way am I pointing?), and Control (how do I actuate to follow the plan?).
  • Trace the closed loop: sensors → state estimator → guidance/control laws → actuators → vehicle dynamics → sensors.
  • Explain why a TVC rocket is unstable without active control (thrust behind the CG, no fins doing the work).
Primer Every vehicle that steers itself — a quadcopter, a re-entry capsule, an orbital booster — runs the same three-function loop, and this course is built around learning that loop by building one. Navigation answers "where am I and which way am I pointing right now," built from noisy sensor measurements. Guidance answers "where should I be," turning a mission objective into a target state or trajectory. Control answers "what do I actuate, right now, to close the gap between the two," and outputs a physical command — in this case, a gimbal angle. A TVC rocket makes the stakes of that loop viscerally clear: with the thrust vector behind the center of gravity and no fins to provide passive aerodynamic restoring torque, the vehicle is an inverted pendulum balanced on a jet of fire. Left alone, any tiny perturbation grows without bound. Everything from Day 2 onward is in service of closing this loop: first the math to describe attitude and motion (Phase 1), then a controller and estimator that close it in simulation (Phase 2), then the same software running for real on a microcontroller (Phase 3).

Do

  • Watch the BPS.space Scout F engineering video end-to-end; list every subsystem you see and which GNC function it serves.
  • Draw a block diagram of the full GNC loop for a TVC vehicle. Label signals with units (rad, rad/s, m/s², PWM µs). You will refine this diagram all course.
  • Set up tools: MATLAB/Simulink + Aerospace Blockset, Arduino IDE, Git repo with folders /notebook, /sim, /flight-software, /report.
